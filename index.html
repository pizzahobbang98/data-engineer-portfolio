부동산 실거래가 데이터 파이프라인 구축

개인 프로젝트 | GitHub Repository

문제 정의

공공 부동산 실거래가 데이터는 조회 자체는 가능하지만, 누적 저장과 정제 구조가 없으면 장기 추이 분석이나 재사용이 어렵습니다. 단발성 조회 중심 구조를 반복 가능한 데이터 자산 형태로 바꾸는 것이 목표였습니다.

해결 방식

  • Open API 기반 수집 로직을 작성해 반복 가능한 수집 환경 구축
  • 결측치 처리, 타입 변환, 컬럼 정리를 통해 Raw Data를 분석 가능한 형태로 정제
  • Python 중심으로 수집부터 적재까지 이어지는 ETL 파이프라인 자동화 구현

왜 이 방법을 선택했는가

이 프로젝트의 핵심은 복잡한 모델보다도, 데이터를 안정적으로 다시 쓸 수 있는 구조를 만드는 것이었습니다. 그래서 빠르게 검증 가능하고 유지보수가 쉬운 Python 기반 파이프라인 형태를 선택했습니다.

향후 개선 방향

현재는 배치성 수집과 정제 중심이므로, 이후에는 지역별 가격 변동을 질의하는 자연어 기반 질의 시스템이나, 뉴스·금리 데이터를 함께 연결한 부동산 시장 분석 Agent로 확장할 수 있다고 생각합니다.

AI Styler Go | 최우수상 수상

문제 정의

여행 중 옷 관리 불편은 분명 존재하지만, 이를 어떤 상황에서 얼마나 자주 느끼는지 정리된 데이터는 부족했습니다. 사용자의 실제 불편을 추정이 아닌 텍스트 데이터 기반으로 구조화하는 것이 필요했습니다.

해결 방식

  • 블라인드 게시물·댓글·대댓글 포함 약 3,000건 크롤링
  • Selenium 기반 스크롤·탭 전환으로 동적 페이지 데이터 수집
  • 네이버 블로그·카페 데이터를 추가 수집해 pain point와 needs를 보강
  • 텍스트 빈도 분석, 워드클라우드, 의도 기반 키워드 그룹화로 사용자 니즈 구조화

왜 이 방법을 선택했는가

초기에는 설문 기반 접근도 가능했지만, 실제 사용자의 언어와 표현을 더 풍부하게 확보하기 위해 공개 커뮤니티 텍스트를 선택했습니다. 특히 다양한 상황 맥락을 얻기 위해 단순 리뷰보다 게시글·댓글 구조가 적합하다고 판단했습니다.

성과와 개선 방향

데이터 기반으로 여행 상황의 의류 관리 니즈를 정리해 휴대용 의류 케어 솔루션 "AI Styler Go" 컨셉을 도출했고, 프로젝트 최우수상을 수상했습니다. 이후에는 키워드 분석을 넘어서 감성 분석, 문맥 분류, 요약 Agent를 적용해 VOC를 자동 분류하고 제품 기획 인사이트를 실시간으로 제공하는 방향으로 발전시킬 수 있습니다.

사용자 리뷰 데이터 기반 UX 개선

문제 정의

사용자 리뷰는 많지만 그대로는 잡음이 많아 인사이트로 바로 활용하기 어렵습니다. 특히 광고성 문구, HTML 태그, 이모지, URL, 중복 표현이 섞여 있어 고객 맥락을 정확히 파악하기 어렵다는 문제가 있었습니다.

해결 방식

  • 오늘의집 커뮤니티 데이터 약 40,000건 수집 및 전처리
  • 결측치 제거, 불필요 컬럼 삭제, 광고성 게시글 필터링, 출처별 병합 수행
  • HTML 태그, URL, 특수문자, 이모지 제거 및 개인정보 마스킹 처리
  • 유니코드·공백 정규화와 맞춤형 불용어 사전 적용으로 텍스트 품질 개선
  • TF-IDF, K-means, LDA 기반으로 고객 맥락을 분석하고 개인화 UX 방향 도출

왜 이 방법을 선택했는가

리뷰 데이터는 모델 성능보다 전처리 품질이 결과를 크게 좌우한다고 판단했습니다. 그래서 먼저 데이터 품질을 높인 뒤, 해석 가능한 방식으로 주제를 묶고 맥락을 찾는 분석 흐름을 구성했습니다.

향후 개선 방향

향후에는 군집과 토픽 모델링 결과를 LLM과 결합해, 사용자의 후기 입력 시 자동으로 불편 유형을 분류하고, 적절한 UX 개선 가이드를 제안하는 리뷰 해석 Assistant 형태로 확장하고 싶습니다.
